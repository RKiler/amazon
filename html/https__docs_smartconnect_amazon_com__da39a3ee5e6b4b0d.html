This guide will help you build an Amazon Yojaka Connector to connect a seller’s ERP/WMS/POS system with the Amazon Yojaka product using a set of RESTful APIs. Apart from the Amazon Yojaka API documentation, sections in this guide will describe how to on-board with Amazon Yojaka, how to authenticate your connector and secure API calls made to Amazon Yojaka, and how to enable sellers to authorize your connector service to invoke Amazon Yojaka APIs on their behalf.

How to get help¶

If you have any questions about Amazon Yojaka, how to develop a connector, or questions about any section in this guide, reach out to us via email at smartconnect-support@amazon.com.

What is a connector?¶

A connector is a service or a library developed to integrate an inventory management and/or order processing system such as a Warehouse Management System (WMS or ERP system) or a Point-of-Sale (POS) system with Amazon Yojaka using Amazon Yojaka’s APIs.

It is the connector’s responsibility to handle transformations and mapping between the WMS/ERP/POS system’s entities and process flows with those supported by Amazon Yojaka.
